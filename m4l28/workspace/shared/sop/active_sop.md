# 产品设计 SOP

## 适用场景
用于新功能或业务流程的产品规格文档设计任务。

## 执行步骤

1. **需求理解**：阅读 `/mnt/shared/needs/requirements.md`，理解目标、边界、约束与风险。
2. **文档撰写**：依据需求，由 PM 撰写产品规格文档，保存到 `/mnt/shared/design/product_spec.md`。
   - 文档须包含：功能概述、用户流程、核心字段定义、边界条件
3. **[CHECKPOINT]** 设计文档完成后，向 Manager 发送完成通知（task_done），等待人工确认。
4. **迭代**（如人工确认返回修改意见）：根据反馈修改文档，重新触发 Checkpoint。

## 角色分工
- Manager：任务分配、SOP选取、最终验收
- PM：产品文档撰写
- **人类甲方**：需求确认（节点1）、设计确认（节点2）——通过 Manager 统一沟通
